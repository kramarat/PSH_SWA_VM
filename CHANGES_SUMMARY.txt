                    PUSH_SWAP CHANGES SUMMARY (APRIL 2026)
================================================================================

1) LIBFT RESTORED
--------------------------------------------------------------------------------
- Reintroduced the bundled `libft/` directory in the build.
- Root `Makefile` now builds `libft.a` first and links it into `push_swap`.
- The project uses `ft_split`, `ft_strlen`, `ft_memset`, `ft_strncmp`, and other libft helpers again.


2) ROOT-LEVEL REPLACEMENTS REMOVED
--------------------------------------------------------------------------------
- Removed the root-level `ft_split.c` and `string_utils.c` replacements.
- `parsing.c` now uses the libft `ft_split` implementation.
- `bench.c` now formats output with libft write helpers.


3) HEADER / USAGE UPDATES
--------------------------------------------------------------------------------
- Updated `push_swap.h`:
  - Includes the bundled `libft/libft.h`.
  - Uses a distinct stack node tag to avoid conflicts with `t_list`.
- Updated `parsing.c`:
  - Uses `ft_strncmp` + `ft_strlen` for flag matching.
  - Uses an internal overflow-safe number parser.
- Updated `Makefile`:
  - Restores the libft sub-build and links `libft.a`.


4) DOCUMENTATION UPDATED
--------------------------------------------------------------------------------
- Updated `README.md` and the text guides to reflect the restored `libft` dependency.
